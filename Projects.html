protocol (OLSR)

A serverless cloud resume website

  • C++
  • Git, GitHub, GitHub Actions
  • OLSR, NS3,

The Ad Hoc network is a wireless network that may not require any infrastructure, defined by a set of mobile nodes, these nodes are mobile and have communication by wireless transmission deployed in a geographically large area to collect and transmit information autonomously. Research on Ad Hoc networks allows us to understand our working environment in this area, respect its aspects (features, topology, applications, etc.). In Ad Hoc network, communication between nodes is based on the ability of collect data and route it to its destination. But guaranteeing access to information depends on routing stability and correct routing information means good quality of the service.

For the smooth running of our work and as part of guaranteeing good quality of Ad Hoc network services, we offer a proposal to guarantee the stability of the link by including distance parameters in the reactive protocol. The aim of this work is to propose a developed OLSR (OLSR-Dev) to achieve better performance than classic OLSR (OLSR-Class). Simulation results explain the effectiveness of the proposed OLSR-Dev. The simulator used to validate the work is the NS-3 simulator. The results show that OLSR-Dev has an improvement of 51% compared to to the OLSR-Class.

SuperMarket Desktop application

An API-based Python GUI app

  • C#
  • Git, GitHub, GitHub Actions
  • ADO.NET

Building a desktop application for a supermarket with C# involves integrating various functionalities to streamline operations

Building a desktop application for a supermarket with C# involves integrating various functionalities to streamline operations, enhance customer experience, and optimize business processes. By focusing on user-friendly design, robust features, and efficient performance, you can create a comprehensive solution tailored to the specific needs of the supermarket environment.

Some notable challenges:

  • Image resizing: I wanted to include the Pixe.la logo as part of the app's header, but when I defined the desired width and height of the image, only a small corner of the full image was shown in the app window. Some searching led me to discover that guizero required the Pillow library to scale images. Installing the Pillow package created the desired outcome.
  • Error handling: the Pixe.la API requires the 'date' header to follow a specific format, so I needed to both convert the date string entered into the app, and catch other errors before data was passed to the API. I learned how to use a try/except block to catch invalid entries, and used the datetime module and strftime to convert the date entry into an appropriate date object.
  • GUI operation: I wanted the app to have a reset button to clear text boxes and reset radio button selections. Research revealed that guizero did not include wrappers for this type of functionality, but that I could use tkinter methods with guizero to solve this challenge. In the case of resetting radio buttons, this involved creating a list of the options and calling tk.select() on the desired list index.
  • Including the graph: Pixe.la graphs are rendered in SVG format in a browser window. I attempted to figure out how to display a graph in the GUI window, only to discover that there doesn't seem to be a Python library that can handle SVG code effectively without relying upon a separate image processing application. I did not like the idea of having to include a separate program with this app, so I settled for a button that opens a link to the selected graph in a new browser tab.

For the time being, this application is complete. Future improvements may include: creating an executable using PyInstaller; adding the ability to register new users and create new graphs; displaying a graph in the app window if a library for doing so becomes available.
